1.2 I部 情報リテラシの現状

1.2.1 1章 ITへの過剰依存が招いた危惧すべき実情

  1. 安易なコピペで失う思考力という大きな代償
  2. 統計学を知らずに ∑ （合計）関数を覚える危険性
  3. 見た目だけでグラフの種類を選ぶ愚かしさ
  4. 身内まで煙に巻いてしまう紙芝居プレゼンテーション
  5. 自らの人格を破壊するモラル不在の匿名掲示板

1.2.2 2章 情報リテラシ教育をとりまく現状

  1. 情報の洪水に流されて自主性をなくした人々
  2. IT 化の進展と引き換えに進む社員の能力低下
  3. ハウツー重視の知識教育が地アタマ力を使う思考力を失わせる

1.2.3 3章 情報リテラシとは何か

  1. ITリテラシと情報リテラシとの違い
    1. パソコン操作を学ぶ前に文書作成の作法を学べ
    2. ITを使いこなす能力と情報を使いこなす能力は別もの
  2. メディア・リテラシと情報リテラシの違い
  3. 「読み書き算盤」にみる情報リテラシの原点
  4. データと情報，知識の違いと関係を理解する
  5. 情報リテラシにおける六つの能力領域

1.3 II部 情報リテラシの体系

1.3.1 4章 情報の収集 / 必要となる情報の入手先を探しだし，利用可能なデータを選別する

  1. 極端意見と中間意見を集めて情報収集源の偏りをなくす
    1. 反対意見がない主流意見は洗練不足
  2. 情報の鮮度と品質を見極める
  3. 表記ゆれと関連性を意識したキーワード検索
  4. プッシュ型とプル型，熟読型と速読型の使い分け
  5. 質問と確認を駆使して「バカの壁」を突破する

1.3.2 5章 情報の加工 / 情報を加工して意味のある情報に変換する

  1. 下準備 客観的事実と主観的事実を区別する
    1. 客観的事実と主観的事実という二つの事実
    2. 飾り付けをはずして客観的事実だけを取り出す
    3. 極端な主観的事実を除去する
    4. 客観的事実にみえる主観的事実も存在する
    5. 主観的事実の方が客観的事実よりも意義があることも
  2. 情報の 2S (整理整頓）
    1. 項目分けは情報整理のための不可欠作業
    2. 並べ替えるだけで見えてくる全体の傾向
    3. 着眼点で様変わりするランキング情報
    4. 類似性，関連性をあぶり出す分類キーの切り替え
    5. 顕著な傾向の背後に隠れた変化に注意する
    6. 将来を左右する情報の勢いを見逃さない
  3. 情報の切り捨て 不要な情報を見極める
    1. 情報は多ければ良いというものではない
    2. 論理のすり替えやありえない根拠で煙に巻く「詭弁」
    3. 正常な思考を妨げる権威というわな
    4. マインドマップによる情報の抽象化
  4. 情報の埋め合わせ 抜けたピースを探し出す
    1. 一部分だけを切り出して見せる報道の危険性
    2. 人の人生すら破壊しかねない片面情報
    3. 5W1H で考える情報の完全性と中立立場で考える情報の妥当性
    4. 主語と語尾が曖昧な日本語
    5. 因果関係による欠損値の推測

1.3.3 6章 情報の分析 / 情報がもつ本質を理解し，有用な知識を導出する

  1. 見える化テクニックで見えてくる全体像
    1. ごまかしや詭弁から身を守る見える化の技術
    2. 図にして考えるのがモデリングの基本
    3. Wikiが証明するコメントの視点追加の意義
  2. スライス＆ダイス＆ドリルダウンによる視点の切り替えが気づきを与える
    1. 定形帳票だけを見ていても気づかない変化が起きている
    2. 時間がかかるデータ分析では急速な状況変化についていけない
    3. ピボットテーブルのクロス集計機能を使いこなす
    4. 瞬時に視点を切り替えることができる OLAP の魅力
    5. 地図グラフも簡単な手間で作成できる
  3. データマイニングとテキストマイニングで浮かび上がる隠れたトレンド
    1. 傾向のアタリを見で確認できる Excel 散布図
    2. 複雑で変化が早い現代社会の助けとなる多変量解析
    3. アンケートの回答者自身も知らない回答者に共通する人格と嗜好
    4. 因子分析で見つける隠れた根本要因
    5. テキストマイニングで浮かび上がる用語間の類似性
  4. 因果関係を追いかけて情報の本質を探りだす
    1. 書き手の意図が原因となって読み手に影響を及ぼす
    2. パレート図とフィッシュボーン図で探る主要な因果関係
    3. 見せかけの因果関係，作り上げた因果関係にだまされない
  5. 近視眼的な問題の見落としを防ぐシステム思考
    1. 窓枠を通して見える風景しか見ていないことに気づかない
    2. やぐら目線が知らせるピンチとチャンス
    3. 抽象のはしごを使ってやぐらを上り下りする
    4. 虫の目，鳥の目，魚の目を使い分ける自在の思考法

1.3.4 7章 情報の蓄積 / 学習した知識と既存知識とを結びつけて新たな知識を蓄積する

  1. インターネット依存によって軽視される知識の定着
    1. 知識の断片だけで全体を知ったつもりになる危険性
    2. 知識は本の中にもインターネットにもない
    3. 学校で学んだ数学がなぜ役にたたないのか
    4. 学歴や資格取得が知識そのものとは違う理由
    5. 既存の知識と組み合わせることで新しい知識を修得する
  2. 「分かる」ためには情報の整理整頓が不可欠
    1. 知っているのと分かっているのとでは意味が違う
    2. 知識を定着させるために必要となる共感という感情
    3. 情報の整理整頓の先に知識がある
  3. 新たな知識によって既知の知識の意味も変わる
    1. 本を読み返す度に意味が変わってくるのはなぜか
    2. 知識の組み合わせが上位の知識を創造する
    3. 知識獲得において理系と文系のこだわりは不利になる
    4. 知識ノートとしての EverNote, OneNote の利用価値
  4. 関連知識の量で決まる未知の情報に対する推理力
    1. 一つの分野のマスターが他分野の知識習得をしやすくする
    2. 反対意見，異論に対する傾聴が関連知識の量を増やす
    3. 旅行や趣味，遊びで得られる知識にも意義がある
    4. 知識を増やすための多読

1.3.5 8章 情報の生成 / 得られた知識を説明するための情報を生成する

  1. 不要な装飾ばかりで大事な部分がわからない
    1. 読み手の時間を浪費する作文レポート
    2. 形容詞は情報伝達における諸刃の刃
    3. ベン図で考える範囲の妥当性
    4. 平安時代の吹き抜け屋台にみる省略の力
    5. Yahoo!ニュースや Google ニュースにみる15文字制限下での要約
  2. コピペではすぐにはがれるオリジナルのつなぎ目
    1. 論文の一部を切り出してつないでみても論文にはならない
    2. 提案書に見るコピペの弊害
    3. 不当なコピペと正当な引用との違い
    4. 三段論法，起承転結から学ぶ論理の作り方
  3. 100枚の論文にも一枚の要旨にまとめることができる論理性がある
    1. なぜ長文をかけない，論文が苦手なのか
    2. モデル図で設計する長文
    3. 論文は長文でなければいけないわけではない
  4. 結論から組み立てるプレゼンテーション
    1. 読み手が迷子になる見えない迷路プレゼン
    2. 最後のページまで我慢させる忍耐プレゼン
    3. 話が進むにつれて論理がぶれていく船酔いプレゼン
    4. プレゼンテーションのデザインは結論ページから
    5. パンくずリストに学ぶ論理構成の見える化
  5. 数式に置き換えられない文章構成は論理に欠陥がある
    1. 結論に関係のない無意味な情報が論理をあいまいにする
    2. 多変量回帰式で設計し検証する論理構造
    3. アウトラインプロセッサによる構造設計
    4. 演繹法と帰納法の違いと相互補完

1.3.6 9章 情報の発信 / 情報を必要とする利用者を探しだし，生成した情報を発信する

  1. タイトルを見ても中身がわからず中身を読んでも結論がない文章の氾濫
    1. キャッチフレーズとしてのタイトルを考える
    2. 読み手の論理センスを狂わせる不親切な情報発信
    3. お友達感覚の携帯電話メールが電子メールマナーをなし崩しにする
    4. 話し言葉で発信される電子メールの危険性
  2. この情報は誰に対するものなのかを考えることから始まる
    1. 口で言うほど簡単ではない顧客志向
    2. 最大限の理解を得ようとするのではなく誤解を最小に防ぐ
    3. 聞き手読み手のタイプと関係性によって発信の仕方を変える
    4. ドラマのあらすじに学ぶ途中参加者への気配り
  3. 情報を伝えたい相手が認知してくれなければ意味がない
    1. 自分勝手な思い込みで相手の立場まで決め付ける人々
    2. 主観だけでも客観だけでも伝わらない
    3. 論文作成に必要となる論理展開力と課題設定力
  4. モラルとコンプライアンスを無視したコンテンツに発信資格なし
    1. 質問，反論を許さない独りよがりの無責任発言
    2. 心の痛みがわからない者に差別用語禁止の理由はわからない
    3. 無責任な情報連鎖が起きやすいソーシャルメディアの危険性
    4. コピペで飾られたあってはならない論文
  5. ソーシャルメディアのプライバシー保護は自己責任
    1. 情報共有が前提のソーシャルメディア
    2. デフォルト設定のままではプライバシーは保護されない
    3. 画像逆検索で暴露されるプライバシー
    4. 安易な友達連鎖が信頼崩壊を生み出す

1.4 III部 情報リテラシの今後

1.4.1 10章 IT 重視の情報リテラシーは何をのこしたのか

  1. 対面，口頭でのコミュニケーションを避ける現代人
    1. IT がなければ仕事ができない社員の急増
    2. 入力の仕方だけ学んでも仕事はできない
    3. 正しい日本語ができないままで外国語を使いこなすことはできない
    4. インターネットとオフィス中心の IT リテラシからの脱却
  2. 陳腐化する情報技術を後追いするだけの情報リテラシ教育
    1. バージョンアップのたびに仕事ができなくなる
    2. 新機能も用途がわからなければ意味がない
    3. 10年前, 20年前とさほど変わらないパソコン利用
    4. IT 弱者に必要なのは IT アレルギーの解消と原理教育
  3. 教育の IT と IT の教育とが区別できていない教育現場
    1. 「教育のIT」で浮かび上がる「ITの教育」の重要性
    2. 教育革命とともにコピペ論文を生み出す「教育のIT」の功罪
    3. 「教育のIT」は教える側への「ITの教育」も必要とする
    4. 「教育のIT」に必要なのは IT の使い方ではなく IT との付き合い方
  4. 「読み書きそろばん」の力を落としかねない IT 教育のジレンマ
    1. 全世代に見られる読み書きそろばん能力の低下
    2. 手書き機会現象によるボキャブラリの貧弱化
    3. 電子メールに代表される書き言葉の話し言語化
    4. 計算式との交換で失った筆算の計算プロセス
  5. IT重視の情報リテラシ教育がもたらしたもの
    1. 無から有を生み出す想像力の喪失
    2. 意見衝突を通じて成長しあう対話力の喪失
    3. 相手の立場になって考える想像力の喪失
    4. 季節の変化や人情の機微を感じ取る感性の喪失
    5. アルファの存在に気づく分析力の喪失

1.4.2 11章 情報リテラシ教育のあるべき姿

  1. 情報のマネジメント・サイクル（インプット，プロセッシング，アウトプット）から学ぶ
    1. 情報発信は形のないものづくり
    2. 新鮮な素材と正しいレシピ，そして情報リテラシ
    3. わがままの押し付けか，思いやりのもてなしか
    4. アウトプットに対するフィードバックというインプット
    5. 気づき合い，学び合いのマネジメント・サイクル
  2. 情報が持つ意味を考える地アタマ力を養う
    1. 抽象的な言葉には無数の具体的な意味がこめられている
    2. 具体的な言葉には共通する抽象的な意味が横たわっている
    3. 一つ一つの言葉の意味を噛みしめるように確かめる
    4. 考えてから話す，聞いた話を咀嚼する
  3. ITに偏らない多様な情報対応力を身につける
    1. 対面だからこそできる真摯に聞くこと
    2. 体験してはじめてわかる人の喜びや悲しみ
    3. 人間力にこだわるリッツカールトンとディズニーランドの凄味
    4. なんの偏見もなく自分の頭で答えを出すという難しさ
    5. 限界を知っているからこそ使いこなせるIT
  4. 情報発信における品質保証と文責意識を形成する
    1. 言いっぱなし，聞き流しがもたらす思考力の喪失
    2. いい加減な伝聞に振り回される人々
    3. 極大と最大を区別しない予言予測のいい加減さ
    4. 発生確率を表現する話し方
    5. 自分の発信に責任を持つという意味
  5. ハウツーではなく考え方重視による陳腐化しない IT リテラシへ
    1. ITリテラシ教育がITリテラシを低下させるという矛盾
    2. ハウツー重視の教育が抱える限界と問題
